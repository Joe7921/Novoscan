# 代码优化工作流

// turbo-all

## 步骤

1. **阅读上下文**：先读 `.agents/context.md` 获取项目背景
2. **定位目标**：确认用户要优化的文件或模块
3. **分析现状**：
   - 运行 `npx tsc --noEmit` 检查类型错误
   - 查看文件复杂度和依赖关系
   - 识别潜在的性能瓶颈
4. **制定策略**（向用户说明但不等待确认）：
   - 列出拟进行的优化项
   - 标注可能影响的外部依赖
   - 预估风险等级（低/中/高）
5. **执行优化**（单文件验证后再批量）：
   - 先修改一个文件
   - 运行 `npm run build` 验证
   - 确认无副作用后继续
6. **验证结果**：
   - 运行 `npm run build` 确保构建通过
   - 对比优化前后的代码变化
7. **输出建议**：
   - 给出至少 3 条进一步优化建议
   - 标注每条建议的优先级和预估工作量

## 优化原则

- ❌ 不改变外观和 UI 风格
- ❌ 不修改与当前任务无关的代码
- ✅ 保持向后兼容
- ✅ 优先消除类型安全问题
- ✅ 减少不必要的重渲染
- ✅ 合并重复逻辑

## 报告漏洞

如果你发现了安全漏洞，**请不要**通过公开的 GitHub Issue 报告。

请通过以下方式联系我们：

📧 **邮件**：[contact@novoscan.cn](mailto:contact@novoscan.cn)

请在报告中包含以下信息：

1. **漏洞描述**：简要描述漏洞类型和影响
2. **复现步骤**：详细的复现方法
3. **影响范围**：可能受影响的用户/数据/功能
4. **修复建议**：如果你有修复方案，请一并提供

## 响应流程

1. 我们会在 **48 小时**内确认收到你的报告
2. 在 **7 个工作日**内提供初步评估
3. 修复后会及时发布安全更新并公开致谢（除非你要求匿名）

## 安全最佳实践

自部署 Novoscan 时，请注意：

- ⚠️ **永远不要**将 `.env.local` 提交到 Git 仓库
- ⚠️ **永远不要**在客户端暴露 `SUPABASE_SERVICE_ROLE_KEY`
- ✅ 定期轮换 API 密钥
- ✅ 使用 HTTPS 部署生产环境
- ✅ 配置适当的 CORS 策略

感谢你帮助保护 Novoscan 社区的安全！
